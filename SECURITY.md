# Security Policy

## Supported versions

Security fixes are applied to the default branch (`main`) of this repository. There are no separate long-term support release lines at this time.

## Reporting a vulnerability

**Please do not open a public issue** for an unfixed security vulnerability. That can put users at risk and makes coordinated disclosure harder.

Instead:

1. Open this repository on GitHub.
2. Go to the **Security** tab.
3. Choose **Report a vulnerability** (private reporting).

If **Report a vulnerability** does not appear: stay on the **Security** tab and open **Advisories** — GitHub’s options there depend on your permissions (reporters often submit through the same **Security** tab flow). As a last resort, contact the repository maintainers privately (for example via the email or site on their GitHub profile).

Include as much as you can:

- Description of the issue and its impact
- Steps to reproduce, or proof-of-concept, if possible
- Affected areas (e.g. build, runtime, optional contact form / Web3Forms integration)
- Versions or commit SHAs if relevant

## What to expect

Maintainers will acknowledge valid reports when they can and work toward a fix. Timelines depend on severity and capacity. Researchers who follow this process will be credited in the advisory or release notes unless they prefer to remain anonymous.

## Scope

In scope for reports:

- This repository’s application code, configuration, and documented build/deploy path
- Supply-chain or dependency issues that materially affect the built site or developer workflow

Out of scope (examples):

- Theoretical issues with no practical exploit path
- Social engineering or spam
- Third-party services (e.g. hosting, Web3Forms) — report those to the provider when appropriate

Thank you for helping keep **Wip3out R3act** and its users safe.
